# PRD — X

## Goal

Use X for short-form, high-frequency signal: punchy takes, threads that unpack a single idea, and replies/quote-tweets that engage the accessibility/AI builder community.

## Specifications

- ≥ 5 posts drafted per week, staged in `content/queue/x/YYYY-MM-DD-<slug>.md`.
- Mix: ≥ 1 thread/week (5–9 tweets), rest standalone posts.
- Each tweet: ≤ 280 chars, no thread-marker emoji (no 🧵), no link in tweet 1 of a thread.
- Threads: each tweet stands alone but threads coherently. Final tweet has a clear takeaway or question.
- Drafts include the suggested posting time slot (morning vs afternoon).

## Forbidden actions

- Never auto-publish.
- Never reuse LinkedIn body text — X requires a fresh structure.
- Never include hashtags unless the user specifically requests one (X-native style).
- Never write engagement-bait ("RT if you agree", "Comment AGREE below").

## Voice

Sharp, opinionated, slightly informal. First-person. One idea per tweet. Threads earn their length — if it can be one tweet, it is one tweet.
